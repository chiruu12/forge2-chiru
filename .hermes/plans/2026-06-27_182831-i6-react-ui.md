# I6 — React UI (Hero) Implementation Plan

> **For Hermes / Coder:** Sprint 2 hero issue. Build the PulseDesk SPA on the existing
> Vite+Tailwind frontend. Design-first via Stitch MCP, then implement in React 19.
> Branch `feat/s2-react-ui`.

**Goal:** Ship a working single-page app — login/register, ticket list+board with filters & search, ticket detail with comment thread, and a dashboard with metrics — all wired to the live PulseDesk API via TanStack Query.

**Architecture:** Feature-first `src/features/<name>/` mirroring the API. A central `apiFetch` wrapper reads `VITE_API_URL` and attaches the Sanctum Bearer token. TanStack Query `QueryClient` for server state, `AuthContext` for the token + current user. React Router v7 for navigation with a protected-route wrapper.

**Tech Stack:** React 19, Vite 8, Tailwind v4 (CSS-first `@import "tailwindcss"`, no JS config), TanStack Query v5, React Router v7, oxlint.

## API Contract (source of truth — from Sprint 1)

Base URL: `VITE_API_URL` (default `http://127.0.0.1:8000`). All protected routes need `Authorization: Bearer <token>`.

| Method | Path | Auth | Body | Returns |
|--------|------|------|------|---------|
| POST | `/api/register` | none | `name, email, password, organization_name, organization_slug` | `201 { token, user }` |
| POST | `/api/login` | none | `email, password` | `200 { token, user }` (422 on bad creds) |
| GET | `/api/me` | sanctum | — | `{ user...+organization }` |
| GET | `/api/tickets` | sanctum+tenant | — | `{ data: Ticket[] }` |
| POST | `/api/tickets` | sanctum+tenant | `subject, description?, status, priority, assignee_id?, requester_id?` | `201 { data: Ticket }` |
| GET | `/api/tickets/{id}` | sanctum+tenant | — | `{ data: Ticket }` |
| PUT | `/api/tickets/{id}` | sanctum+tenant | any subset of create fields | `{ data: Ticket }` |
| DELETE | `/api/tickets/{id}` | sanctum+tenant | — | `204` |

### Ticket shape (JSON)
```jsonc
{
  "id": 1,
  "organization_id": 2,
  "subject": "Login button broken",
  "description": "Steps to reproduce...",   // text|null
  "status": "open",        // open | in_progress | pending | resolved | closed
  "priority": "high",      // low | medium | high | urgent
  "requester_id": 5,
  "assignee_id": 3,        // null if unassigned
  "created_at": "2026-06-27T12:00:00.000000Z",
  "updated_at": "2026-06-27T12:30:00.000000Z",
  "requester": { "id": 5, "name": "Jane", "email": "...", "organization_id": 2, "role": "customer" },
  "assignee":  { "id": 3, "name": "Bob",  "email": "...", "organization_id": 2, "role": "agent" }  // or null
}
```

### User shape
```jsonc
{ "id": 5, "name": "Jane Doe", "email": "jane@x.com", "organization_id": 2, "role": "admin", "organization": { "id": 2, "name": "Acme", "slug": "acme" } }
```

### Role enum
`admin` | `agent` | `customer` — customers only see their own tickets (enforced server-side).

## Phase 2 — Core Infrastructure

### Task 2.1: Central API client
**Create:** `frontend/src/lib/apiFetch.js`
```js
const BASE = import.meta.env.VITE_API_URL || 'http://127.0.0.1:8000'

export async function apiFetch(path, { method = 'GET', body, token } = {}) {
  const headers = { 'Accept': 'application/json' }
  if (token) headers['Authorization'] = `Bearer ${token}`
  if (body) {
    headers['Content-Type'] = 'application/json'
    body = JSON.stringify(body)
  }
  const res = await fetch(`${BASE}/api${path}`, { method, headers, body })
  if (!res.ok) {
    const error = new Error(`API ${res.status}`)
    error.status = res.status
    try { error.body = await res.json() } catch { error.body = null }
    throw error
  }
  if (res.status === 204) return null
  return res.json()
}
```

```js
import { useQuery } from '@tanstack/react-query'
import { fetchTickets } from './api'
export function useTickets(token) {
  return useQuery({ queryKey: ['tickets'], queryFn: () => fetchTickets(token) })
}
```

### Task 4.2: Ticket API wrapper
**Create:** `frontend/src/features/tickets/api.js`
```js
import { apiFetch } from '../../lib/apiFetch'
export const fetchTickets  = (token)               => apiFetch('/tickets',         { token }).then(r => r.data)
export const fetchTicket   = (token, id)           => apiFetch(`/tickets/${id}`,   { token }).then(r => r.data)
export const createTicket  = (token, body)         => apiFetch('/tickets',         { method:'POST', body, token }).then(r => r.data)
export const updateTicket  = (token, id, body)     => apiFetch(`/tickets/${id}`,   { method:'PUT', body, token }).then(r => r.data)
export const deleteTicket  = (token, id)           => apiFetch(`/tickets/${id}`,   { method:'DELETE', token })
```
